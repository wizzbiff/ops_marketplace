# Report Style, the shared house standard

Tone and formatting rules shared by every report this org produces, in any domain. Domain plugins reference this skill rather than restating the rules, so a change here updates the voice of every report at once.

## Voice

* Plain and factual. No filler, no praise, no hedging.
* Lead with what the reader must act on. Detail comes after.
* Short over complete. A report nobody finishes is a report nobody trusts.

## Substance

* Name items by their identifier and a short title so anyone can jump straight to them.
* Never include per person performance framing in any leadership facing section. Reports describe flow of work and risk, not individuals.
* If a data source failed or the data is incomplete, say so at the top rather than presenting a confident but partial picture.
* If anything material was guessed, flag it at the top.

## Formatting

* Markdown. Headings for sections, short bullets for items.
* Numbers up front in summaries.
* Omit empty sections rather than printing a placeholder for each, unless a specific report template says otherwise.
